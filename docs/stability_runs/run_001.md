# Stability Run 001 — Phase 4 baseline

**Date:** 2026-04-14
**Backend:** lavapipe (Mesa software Vulkan, CPU device)
**Build:** debug profile
**Branch:** `claude/start-iris-implementation-HEwTw` (Phase 4)

## Setup

First end-to-end stability validation after wiring Phase 4's continual-learning
mechanisms (replay mixing, representation drift monitor, entropy floor) into
the agent. Runs are short (5k steps) because lavapipe is CPU-only — meaningful
1M-step runs are reserved for real GPU.

## Observations

- **No NaN/divergence on any environment.** Numerical clamping in the credit
  history input (`[-5, 5]`) and the advantage skip-on-negative policy gradient
  rule kept everything bounded.
- **World model converges everywhere.** Loss drops 1–3 orders of magnitude
  within 5k steps in all envs. The encoder learns a compact representation
  even from sparse one-hot inputs (Taxi).
- **Drift monitor stays cold.** With the encoder LR-scaling backstop in place,
  drift stays under 0.25 across all runs. The reactive LR-reduction was never
  triggered in these short runs.
- **Replay mixing works.** `loss_replay` is consistently within an order of
  magnitude of `loss_world_model`, indicating the encoder generalizes to past
  samples without catastrophic forgetting.
- **Policy entropy holds.** Discrete entropy stays at the uniform-distribution
  ceiling (`ln(num_actions)`) — expected since intrinsic-only rewards rarely
  produce strong action preferences in 5k steps. The entropy floor never
  triggered to suppress updates.
- **Credit assigner sensitivity.** Of all modules, the credit assigner is the
  most sensitive to input scale. The reward-normalization in `flatten_history`
  combined with the `[-5, 5]` clamp is what keeps Acrobot/Taxi from NaN-ing
  the way they did before Phase 4.

## Per-env LR notes

- **Default `learning_rate=1e-3`** is fine for ≤5-dim observations.
- **Wider observations need lower LR.** Acrobot (6-dim) → `1e-4`. Taxi
  (34-dim) → `3e-5`. The Xavier init's effective scale doesn't perfectly
  compensate for input dimensionality on lavapipe with single-sample updates.
- A future `Adam` optimizer in meganeura would likely tolerate a uniform LR.

## Known issues to revisit

1. **Policy loss magnitudes are large** on Acrobot/Taxi/Pendulum (10–50).
   This is because we don't normalize the policy loss by the value-target
   magnitude. The policy *does* train (loss decreases over time), but
   absolute values aren't directly comparable across envs.
2. **`H_eff` is stuck near history midpoint** (e.g. 3.5 for H=8, 7.5 for H=16).
   The credit assigner hasn't yet learned to bias attention temporally.
   This needs longer runs and stronger reward differentials to develop.
3. **`policy_entropy` for continuous = exploration std**, not true entropy.
   A learned-variance Gaussian would expose actual entropy here.

## Next checkpoints

- Wire `Adam` once meganeura exposes it, retest LR robustness.
- Phase 5: lift the frozen reward circuit if `H_eff` grows past a threshold.
